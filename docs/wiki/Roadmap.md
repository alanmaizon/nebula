# Roadmap

Roadmap phases align with `DEVELOPMENT_PLAN.md`.

## Step 0: Governance and Bootstrap
- Templates, policy, and automation in place.

## Step 1: Foundation
- Backend/frontend scaffolding, env setup, and local startup path.

## Step 2: Ingestion
- Upload + parse + metadata persistence.

## Step 3: Retrieval
- Chunking, embeddings, and evidence retrieval.

## Step 4: Requirements Extraction
- Validated `requirements.json`.

## Step 5: Cited Drafting
- Evidence-grounded draft generation.

## Step 6: Coverage
- Requirement coverage matrix and missing evidence list.

## Step 7: Export and UX
- JSON/Markdown export and demo-ready flow.

## Step 8: Hardening
- Security controls, observability, and backup/restore documentation.

## Step 9: Release
- Demo freeze, quality gate, and release tag.

## Step 10: Nova Multi-Agent Migration
- Replace heuristic intelligence path with AWS Strands + Amazon Nova agent orchestration.
- Preserve existing API contracts while proving Nova usage in submission evidence.
